# 가다 데스크 디자인 가이드

## UI·UX 원칙

이 서비스의 UI/UX는 화려한 그래픽보다 **심플하고 단순한 의미 전달**을 최우선으로 설계한다.

- 사용자가 첫 화면에서 "무엇을 제공하는 서비스인지" 즉시 이해할 수 있어야 한다.
- 불필요한 장식, 과한 애니메이션, 복잡한 배경 패턴은 사용하지 않는다.
- 핵심 메시지, 기능, 요금제, 도입 절차가 명확하게 구분되도록 구성한다.
- B2B SaaS 서비스답게 신뢰감 있고 정돈된 화면을 만든다.
- 인력사무소 사용자가 어렵게 느끼지 않도록 메뉴, 버튼, 카드, 폼 구조를 직관적으로 배치한다.
- 모든 섹션은 **"짧은 제목 + 간결한 설명 + 명확한 CTA"** 구조를 따른다.
- 텍스트는 길게 나열하지 말고, 핵심 문장과 카드형 정보 구조로 정리한다.
- 기능 설명은 전문 용어보다 현장 실무자가 이해하기 쉬운 표현을 우선한다.
- 모바일에서도 정보가 복잡해 보이지 않도록 여백, 줄 간격, 카드 간격을 충분히 둔다.

## 가다 브랜드 컬러

가다의 키컬러를 중심으로 일관된 브랜드 경험을 제공한다.

| 토큰 | HEX | 용도 |
| --- | --- | --- |
| Primary Blue | `#0669F7` | 주요 CTA, 강조 텍스트, 활성 상태, 핵심 아이콘 |
| Secondary Yellow | `#FFC72C` | 추천 배지, 중요 알림, 강조 포인트 |
| White | `#FFFFFF` | 기본 배경 |
| Black / Text | `#111827` (slate-900 계열) | 본문 텍스트 |
| Gray / Sub Text | slate-500 ~ slate-600 | 보조 설명 |
| Background | white, slate-50, blue-50 | 섹션 배경 |

### 컬러 사용 원칙

- **Primary Blue(`#0669F7`)** 는 주요 CTA 버튼, 강조 텍스트, 활성 상태, 핵심 아이콘에 사용한다.
- **Secondary Yellow(`#FFC72C`)** 는 추천 배지, 중요 알림, 강조 포인트에 제한적으로 사용한다.
- 배경은 대부분 흰색 또는 옅은 블루/그레이 계열을 사용해 깔끔하게 유지한다.
- 한 화면에 너무 많은 컬러를 사용하지 말고, **Blue / Yellow / White / Slate** 계열로 제한한다.
- Pro 요금제처럼 강조가 필요한 영역에는 Blue 라인, Yellow 배지, 연한 Blue 배경을 조합한다.
- 버튼은 Primary Blue 기반으로 통일하고, 보조 버튼은 흰색 배경 + slate border 스타일을 사용한다.

## 시각적 방향성

- 전체 디자인은 **"현장 업무를 쉽게 정리해주는 디지털 데스크"** 느낌이어야 한다.
- 복잡한 건설 이미지보다, 깔끔한 대시보드·카드·리스트 UI를 중심으로 표현한다.
- Hero 섹션의 대시보드 목업은 실제 운영 현황을 보여주는 것처럼 단순하고 명확하게 구성한다.
- 요금제 카드는 비교가 쉬워야 하며, 기능 목록은 아이콘과 짧은 문장으로 정리한다.
- 문의 폼은 입력 항목이 많아 보이지 않도록 넓은 여백과 명확한 placeholder를 사용한다.
- 전반적으로 **"쉽다, 명확하다, 믿을 수 있다"** 는 인상을 주는 디자인을 목표로 한다.
